## **1. Overview of Topic**

Heart disease, specifically **cardiovascular diseases (CVDs)**, remains one of the leading causes of death worldwide. Early detection and intervention are critical in reducing the mortality rate associated with these diseases. However, traditional diagnostic methods often require expensive, invasive procedures and are not scalable for large population screenings.

With the advent of **Machine Learning (ML)**, predicting heart disease from clinical data has become more efficient. Machine learning models can analyze vast amounts of clinical data to identify patterns that may not be easily detected by human doctors. This project aims to build a **Machine Learning Pipeline** for heart disease prediction, utilizing various algorithms and advanced techniques like **Explainable AI (XAI)** to improve interpretability and transparency of the predictions.

This project includes:
- Data preprocessing and feature engineering.
- Model training using multiple machine learning algorithms.
- Hyperparameter tuning for better model performance.
- Use of **XAI** techniques (SHAP, LIME, and Permutation Importance) for model explainability.
- Integration of **Groq API** to improve processing speed and efficiency for prediction and explanation.
The goal is to create an accurate, interpretable, and scalable model for heart disease prediction that can be applied in real-world healthcare applications.

## **3. Evaluation and Model Improvement**

The models in this project are evaluated on various metrics like **accuracy**, **precision**, **recall**, and **F1-score**. 

### **Model Performance Comparison**

### **1. No Feature Engineering vs Applied Feature Engineering** 

In the initial stage, models were tested without any feature engineering (No FE). Then, feature engineering techniques were applied to improve the dataset and model performance. The Feature Engineering (FE) in this project specifically involves removing weak entities based on exploratory data analysis (EDA) using boxplots and statistical testing using Welch's t-test. These techniques help in identifying and removing features that do not significantly contribute to the model's predictive power.

- No Feature Engineering: Models trained without any preprocessing or feature transformation.

- Applied Feature Engineering (FE): Models trained after removing weak entities using EDA boxplots and Welch's t-test.

### **2. After Tuning vs After XAI-Guided Improvement** 

### **XAI-Guided Model Improvement Workflow**

1. **Permutation Importance**: 
   - Identify important features by randomly shuffling their values and observing how much the model's accuracy decreases. Features with the most impact are ranked.

2. **Create Interaction Features**: 
   - Based on the important features, new interaction features are generated by combining (multiplying) the values of two or more features. These are added to the original dataset.

3. **Retrain the Model**: 
   - The model is retrained using the same optimized hyperparameters, ensuring any performance changes are due to the new interaction features.

4. **Compare Results**: 
   - The retrained models with **XAI-guided features** are compared to baseline models to assess performance improvements.

### **XAI Explanation for Predictions**

#### **1. Global Explanation**
- **Global explanations** provide insights into the overall behavior of the model by identifying which features are important across all predictions. Both **SHAP** and **Permutation Importance (PI)** are used to evaluate feature importance.

  - **SHAP (SHapley Additive exPlanations)**: SHAP values are used to explain the contribution of each feature to the model's predictions. SHAP assigns each feature a value that represents its impact on the prediction, ensuring the explanation is consistent with the model’s overall behavior. This method is particularly useful for understanding the interaction between features and the model’s decisions.

  - **Permutation Importance (PI)**: PI evaluates the importance of each feature by randomly shuffling its values and observing how much the model's accuracy decreases. Features whose shuffling results in a significant drop in accuracy are considered more important. PI provides a more global view of feature importance, helping to rank features based on their overall contribution to the model’s performance.

Together, **SHAP** and **PI** provide a comprehensive global explanation of feature importance, with **SHAP** focusing on individual feature contributions and **PI** offering an overall view of feature impact across all predictions.

#### **2. Local Explanation**
- **Local explanations** help explain individual predictions by identifying which features were most influential in making a particular prediction. Techniques like **LIME (Local Interpretable Model-Agnostic Explanations)** are used to provide these explanations. LIME works by approximating the model with a simpler, interpretable model around the prediction point to understand which features contribute most to the outcome.
- - When the user fills in input data (such as clinical features) for a prediction, **LIME** is used to explain how each feature in the input affects the model’s decision for that specific data point. LIME generates an explanation that is specific to the filled-in input, helping the user understand the factors contributing to the prediction.

## **5. Final Setup Instructions**

### 1. Install Dependencies**

To ensure that the project runs smoothly, install all necessary Python packages using the following command from your terminal (in your project directory):
   ```
pip install -r requirements.txt
   ```

### 2. Set Up the .env File

- Make sure you have your **Groq API Key** from Groq's API Console.
- Create a `.env` file in the project root directory (if it doesn't exist already).
- Add the following lines to your `.env` file:

    ```
    GROQ_API_KEY=your_generated_api_key
    ```

### 3. Run the Streamlit Application

After setting up your environment, run the app by executing this command:

    ```
streamlit run app.py
    ```

### 4. Interacting with the App

- When the app runs successfully, you'll see a URL in your terminal, usually: 

    ```
    http://localhost:8501
    ```

- Open the URL in your browser to interact with the app.

### 5. Upload Data and Start Training

- In the **Data & Training** section, you’ll be able to upload your heart disease dataset (e.g., `heart_statlog_cleveland_hungary_final.csv`).
- Enable the **XAI-Guided Model Improvement** option, and click **Start Training**.
- The app will begin training the model and you can monitor the progress.
